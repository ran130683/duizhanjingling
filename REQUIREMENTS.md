# 自走棋阵容生成器 — 需求文档

> 本文档汇总截止当前迭代的全部产品需求与实现约束，作为后续维护的依据。

---

## 一、项目定位

一个用于卡牌自走棋游戏的 **轻量级阵容图生成工具**：
- 用户根据每关阵容代码，自动生成 4×5 棋盘阵容图
- 支持英雄 / 宠物 / 职业资源管理
- 单页面应用，浏览器 `localStorage` 持久化，**点开即用**
- 可选本地 Node 服务把导入头像保存到项目目录 `assets/heroes`、`assets/pets`
- 文件结构：`index.html` + `style.css` + `app.js` + `server.js`，外加 CDN 引入 `html2canvas`

启动方式（推荐，可持久化头像文件）：
```bash
cd card-formation && node server.js
# 访问 http://localhost:8080
```

纯静态启动方式（不写项目目录，头像仍存 localStorage/base64）：
```bash
cd card-formation && python3 -m http.server 8080
# 访问 http://localhost:8080
```
（必须用 `http(s)://` 而非 `file://`，否则剪贴板等 API 受限）

## 二、整体功能模块

| 模块 | 用途 |
|---|---|
| 阵容生成 | 输入阵容代码 → 渲染棋盘图 → 复制 / 下载 |
| 手动布阵 | 拖拽英雄到 4×5 棋盘 → 多关卡编排 → 复制阵容图 / 生成阵容脚本 |
| 英雄管理 | 单个增删改、按文件夹批量导入、按职业筛选 |
| 宠物管理 | 单个增删改、按文件夹批量导入 |
| 职业管理 | 增删改 + 自定义颜色 |
| 数据导入 / 导出 | JSON 备份与恢复 |

## 三、数据模型

### 3.1 存储位置
- `localStorage` key：`card-formation-data-v1`
- 主题偏好 key：`card-formation-theme`
- 使用 `node server.js` 启动时，导入 / 保存头像文件会写入：
  - 英雄：`assets/heroes`
  - 宠物：`assets/pets`

### 3.2 数据结构
```js
{
  classes: [{ id, name, color }],         // 职业
  heroes:  [{ id, name, classId, quality, avatar }], // avatar = base64 或 /assets/heroes 路径
  pets:    [{ id, name, quality, avatar }],           // avatar = base64 或 /assets/pets 路径
}
```

### 3.3 容量与压缩
- 所有上传 / 导入的图片**自动压缩**：最大边 200px，WebP/JPEG 0.85
- 使用 `node server.js` 时，压缩后的头像保存为项目目录文件，可显著降低 localStorage 占用
- 纯静态模式下单张头像约 5–15KB，可容纳 ~300 个英雄
- `localStorage.setItem` 失败时**显式弹窗**提示原因（容量满 / `file://` 协议禁用），不静默丢失

## 四、英雄管理

### 4.1 单个增删改
- 输入名称、选择职业、上传头像
- 编辑时若不重新选头像，则保留原头像
- 头像存储格式：base64（已压缩）

### 4.2 批量导入（按文件夹）
- 使用 `<input webkitdirectory>` 选择文件夹（递归扫描）
- **文件名格式**：`职业_英雄名.扩展名`
  - 分隔符兼容：`_`、`-`、空格
  - 例：`战士_鲛女.png` → 英雄「鲛女」+ 职业「战士」
- 职业不存在时**自动创建**，颜色基于名称 hash 稳定生成
- 无前缀的文件名（如 `鲛女.png`）：使用面板下拉的默认职业
- 同名英雄：可勾选「覆盖头像」（默认勾选）
- 进度条 + 完成统计：`新增 X，更新 Y，跳过 Z，自动新建职业 N`
- 支持格式：png / jpg / jpeg / gif / webp / bmp / svg
- 稳定性要求：
  - 不一次性把整个文件夹读成 base64 数组
  - 逐个文件处理，使用 `URL.createObjectURL(file)` 读取原图
  - 每张图片压缩后立即释放 object URL，并让出一次主线程
  - 保存失败必须停止导入并提示容量 / 协议等错误原因
  - 使用真实头像全集文件夹回归测试：`~/Documents/Codex/hero-avatar-crops-fuller/`

### 4.3 按职业筛选 Tab
- 列表上方显示「全部 / 各职业 / 无职业」药丸 Tab
- 每个 Tab 带数量徽章
- 激活态用对应职业颜色高亮（与棋盘边框色一致）
- 联动：增删英雄、删除职业、导入新职业，Tab 实时刷新
- 当前激活职业被删除时自动回退到「全部」

## 七、阵容代码解析

### 7.1 完整格式
```
阵容名 - 前缀@前缀@英雄白名单@阵型代码@@尾缀
```

**示例：**
```
合9法弓套法师牛-8@0@法核|鲛女|先知|双头龙|火灵|雷神|风灵|剑仙|蛮牛|光@1，鲛女511.法核512|47，火灵011.鲛女544@@7
```

### 7.2 切片规则
| 步骤 | 处理 |
|---|---|
| 阵容名 | `-` 之前的字符串 |
| 尾缀 | `@@` 之后的内容直接丢弃 |
| 阵型代码 | 按 `@` 分段，**从右往左**找第一个含「，/,」的段 |
| 英雄白名单 | 阵型代码段之前，第一个含 `|` 但不含「，/,」的段 |
| 容错 | 旧格式（无 `-`/`@`）也兼容，整串当阵型代码 |

### 7.3 阵型代码细则
- `|` 分隔关卡
- 关卡内部：`关数，英雄1.英雄2...`（中英文逗号兼容）
- 每个英雄：`名称 + 3 位数字`
  - 第 1 位：等级，**0 表示下场**
  - 第 2 位：行号 (1–4)
  - 第 3 位：列号 (1–5)
- 关卡间状态**累积**：第 N 关在第 N-1 关基础上叠加变更

### 7.4 白名单校验
- 解析时若出现不在白名单中的英雄名，弹窗警告：
  > ⚠️ 检测到不在英雄列表中的英雄：xxx
  > 本场上场列表：…
  > 请检查阵容代码或在「英雄管理」补充对应英雄。
- 校验只是提示，不会阻止渲染

## 八、阵容生成页面

### 8.1 顶部布局
- **阵容名输入框**（金色实色 + 暗底，可见可改）
  - 默认值来自代码解析的 `阵容名`，无名时填「我的阵容」
  - 修改后会用于：导出图标题、下载文件名
- **棋盘风格切换**：7 种主题
  - 经典（默认）／深绿／蓝白／紫晶／原木／霓虹／暗夜
  - 实时切换，所有已渲染棋盘立即应用
  - 选择持久化到 `localStorage`

### 8.2 棋盘渲染
- 4 行 × 5 列网格
- **国际象棋深浅交替格**（每个主题有自己的浅色 / 深色）
- 棋盘外金属边框 + 阴影
- 英雄方块**贴满整格**（不再是圆头像）
- 英雄信息显示：头像 + 名字底条（**不显示等级 Lv**）
- 英雄边框 = 该英雄所属职业颜色

### 8.3 棋盘网格布局
- 桌面端：固定 **2 列**（一排最多 2 个棋盘，避免手机字小）
- 移动端（≤640px）：单列
- 关卡标题简化为「第 X 关」
- 棋盘整体紧凑：`padding 5px`、外卡片 `padding 10px`、`border 3px`

### 8.4 关卡变化提示
- 第 2 关起，每张棋盘下方显示与上一关相比的差异
- 仅展示两类（**不展示移位**）：
  - **下**（红）：上关在场、本关消失（含 `0` 标记下场）
  - **上**（绿）：本关新出现
- 每个英雄渲染为「小芯片」：圆头像 + 名字 + 职业色描边
- 没有变化的关卡（含第 1 关）不显示提示行

### 8.5 操作按钮
| 按钮 | 行为 |
|---|---|
| 解析渲染 | 解析阵容代码并渲染棋盘 |
| 关卡选择 | 下拉选中要单独导出的关卡 |
| 复制当前关 | 选中关卡截图复制到剪贴板 |
| 复制全部阵容 | 整张大图（含标题）复制到剪贴板 |
| 下载全部阵容 | 整张大图保存到本地 |

## 九、手动布阵页面

### 9.1 页面目标
- 不依赖阵容代码，直接通过拖拽完成阵容编排
- 适合从零搭配阵容、微调站位、反向生成阵容脚本
- 与「阵容生成」共用同一套英雄、职业、头像和棋盘视觉规则

### 9.2 页面布局
- 左侧：多关卡棋盘区
  - 每个关卡是一张 4×5 棋盘
  - 支持新增关卡、删除关卡、清空单关、复制单关阵容
  - 每个关卡可手动设置关卡数，例如第 1 关、第 8 关
- 右侧：英雄池
  - 展示已导入 / 已创建的全部英雄
  - 支持「全部 / 各职业 / 无职业」筛选
  - 英雄头像使用方形卡片，显示头像、名称、职业色
- 桌面端左右分栏，移动端上下堆叠

### 9.3 拖拽规则
- 从英雄池拖动英雄头像到棋盘格，即可放置英雄
- 点击棋盘中的英雄，可从该格移除
- 同一关卡内同一个英雄只能出现一次：
  - 若拖到新格子，会自动从原格子移除并放到新位置
- 棋盘格坐标沿用阵容代码规则：
  - 行号 1–4
  - 列号 1–5

### 9.4 多关卡规则
- 初始至少有 1 个关卡
- 点击「添加关卡」会新增一张独立棋盘
- 新关卡默认关卡数 = 当前最大关卡数 + 1
- 删除关卡时至少保留 1 个关卡
- 各关卡棋盘是独立状态，脚本生成时再转换为阵容解析器需要的「累积变更」格式

### 9.5 阵容脚本生成
- 输出格式使用完整阵容格式：
```
阵容名-0@0@英雄白名单@阵型代码@@0
```
- 英雄白名单来自所有关卡中出现过的英雄名，使用 `|` 连接
- 第一关输出所有上阵英雄
- 后续关卡与上一关对比后输出变更：
  - 新增或移位英雄：`英雄名5行列`
  - 从上一关消失的英雄：`英雄名0行列`
- 手动布阵未设置等级时，默认等级使用 `5`
- 生成后的脚本可：
  - 一键复制到剪贴板
  - 一键发送到「阵容生成」页面解析渲染

### 9.6 手动阵容图导出
- 支持「复制阵容图到剪贴板」
- 导出内容包含阵容名、所有手动关卡棋盘
- 使用与阵容生成页面一致的 `html2canvas` 导出链路
- 复制前需确认生成的 PNG 不是空白图

## 十、图片导出

### 10.1 渲染策略
- 优先使用 `html2canvas` 的 `foreignObjectRendering: true`
  - 用 `<foreignObject>` 内嵌 HTML，**色彩与网页 100% 一致**（解决 P3→sRGB 偏色问题）
- 失败自动回退到普通光栅模式

### 10.2 导出前预处理（`flattenForExport`）
- 移除棋盘格内嵌暗角阴影（`inset box-shadow`），保留 2px 职业色描边
- 移除英雄底部黑色渐变阴影（伪元素 `::before`）
- 关卡标题渐变文字改为实色金 `#ffd166`（避免 fallback 偏暗）
- 棋盘外卡片半透明背景改为实色（避免叠加偏暗）

### 10.3 单关导出
- 临时容器宽度 480px，2 倍 scale
- 复制 PNG 到剪贴板

### 10.4 全部阵容导出
- 临时容器宽度 1000px，2 列固定网格
- 顶部金色实色大标题 = 用户编辑过的阵容名
- 复制 / 下载两种出口，文件名：`{阵容名}-全部阵容.png`（自动过滤非法字符）

### 10.5 剪贴板复制要求
- 导出临时节点必须放在页面内可渲染区域，不能使用远离视口的 `left:-10000px`
- 复制前先把 html2canvas 结果绘制到标准 PNG canvas：
  - 先填充稳定背景色，避免透明背景粘贴异常
  - 再 `drawImage` 写入导出画面
  - 使用 `toDataURL('image/png')` → `fetch(dataUrl)` → `blob` 生成剪贴板数据
- blob 体积过小或像素检测明显为空时，应抛出错误，不允许把空白图复制给用户

### 10.6 反馈
- 操作完成后右下角 toast：
  - 成功：`✅ 已复制到剪贴板` / `✅ 已下载`（绿）
  - 失败：错误信息（红）
- 不再使用 `alert()` 打断

### 10.7 限制
- 复制 API（`navigator.clipboard.write` + `ClipboardItem`）需 `http(s)://` 或 `localhost`
- 第一次复制 Chrome 会请求「剪贴板权限」

## 十五、已知边界

1. **localStorage 容量上限**：超过 5MB 会保存失败 → 已提示用户
2. **`file://` 协议**：剪贴板与部分 API 不可用 → 推荐用本地 server 启动
3. **批量导入手机不支持**：`webkitdirectory` 在移动浏览器多数不支持，移动端走「数据导入 JSON」恢复
4. **复制图片需用户允许剪贴板权限**：首次会有浏览器弹窗
